# OUTFItazo 👕🕹️

**Armario Virtual & Outfit Generator**

Un simulador web interactivo diseñado con una estética arcade y toques retro para gestionar tu ropa y crear *outfits* de manera visual e intuitiva.

## 🚀 Características Principales

- **Outfit Studio (Visualizador 3D):** Combina partes de arriba, chaquetas/sudaderas, pantalones y calzado sobre un maniquí virtual con perspectiva isométrica. Puedes rotar las prendas y ver detalles interactivos.
- **Cajón de Armario Retro:** Un menú dinámico estilo máquina recreativa de los 90 donde tu ropa está catalogada en forma de **"Tazos" coleccionables** con efecto *pixel art* al pasar el ratón.
- **Añadir Prendas (Tag Premium):** Un formulario inmersivo con diseño de etiqueta de boutique de cartón *kraft* para registrar nueva ropa en tu colección digital.
- **Diseño Inmersivo:** Interfaz de cristal (*glassmorphism*), texturas de madera y papel, efectos 3D de profundidad, sombras realistas y microinteracciones muy pulidas.

## 🕹️ Cómo Funciona

1. **Tu Colección:** En la pestaña **Mi Armario**, abre el cajón de madera para ver toda tu ropa guardada. Al hacer clic en cualquier tazo de prenda, esta "volará" y se equipará automáticamente en el maniquí del Outfit Studio.
2. **Outfit Studio:** Previsualiza cómo quedan las prendas juntas. Haz clic en el botón de rotar para dar la vuelta a la ropa, o mantén el ratón sobre cualquier prenda para ver su descripción en un tooltip flotante.
3. **Gestión:** Usa la gran etiqueta de ropa para añadir prendas nuevas. Solo necesitas rellenar los huecos (categoría, color, estilo) y subir una foto de tu prenda.

## 🛠️ Tecnologías Utilizadas

Desarrollado puramente con tecnologías web nativas (Frontend estático, sin frameworks):
- **HTML5** para la estructura semántica.
- **CSS3 (Vanilla)** para todo el diseño complejo: animaciones fluidas, filtros *pixel art*, transformaciones 3D, sombras compuestas y diseño responsivo.
- **JavaScript (Vanilla)** para la lógica de visualización, interactividad de los tazos y gestión del DOM.

Al ser puramente frontend estático, no requiere instalación de dependencias, base de datos ni servidores backend.
